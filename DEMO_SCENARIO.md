# 경영진 시연 시나리오 — 그룹웨어 MVP

- **목적**: v0.9.33 스펙의 그룹웨어 기능이 실제 동작 가능함을 경영진에게 시연
- **소요**: 약 8~10분 · **데이터**: 프론트 mock (서버 불필요)

## 사전 준비
```bash
cd frontend && npm install && npm run dev
# 브라우저에서 http://localhost:5173 접속
```

## 시연 흐름

### 1. 로그인 · 권한 체계 (2분)
1. 로그인 화면에서 **"마스터 (대표이사)"** 데모 버튼 클릭 → 자동 로그인.
2. 좌측 사이드바에 **6개 그룹 15개 메뉴 전체**가 보임을 확인 (마스터 권한).
3. 로그아웃 후 **"일반 (홍길동)"** 로그인 → 인사·근태/수금관리/회원관리 메뉴가 **숨겨짐**을 시연.
   → *부서별 접근 권한(MENU_PERMS)이 실제로 동작함을 강조.*

### 2. 대시보드 (1분)
- 총 계약금액·미수금 총액·결재 대기·휴가 승인 대기·환경 성적서 현황이 **집계 카드**로 표시.
- 최근 전자결재/공지사항 요약 위젯.

### 3. 핵심 업무 — 사업관리 (2분)
- **계약·미수금관리**: 통합계약 / 미수금현황 / 청구·수금 탭 전환. 공동계약 배지, 진행률 바, 미수금 강조.
- **환경질현황**: 발주처코드+연도로 **관리번호 자동 채번**(`2026-CL001-002`) 시연 → "채번" 버튼 클릭.

### 4. 인사·근태 — 살아있는 입력 (2분)  ※ 마스터/경영지원 계정
- **휴가관리**: 근속연수 기반 **근로기준법 연차 자동 계산** 카드 확인.
- **"휴가 신청"** 버튼 → 모달에서 유형/기간 입력 → 신청 → **목록 상단에 즉시 추가 + 토스트 알림**.
  → *읽기 전용이 아니라 실제 입력/처리 흐름이 구현됨을 강조.*

### 5. 수금·장비 (1분)
- **거래처등록**: "거래처 등록" 모달로 신규 거래처 추가(코드 자동 부여) → 목록 반영 + 토스트.
- **장비관리**: 장비 대장 / 이상 보고서 탭.

### 6. 관리자 기능 (1분) ※ 마스터 전용
- **회원정보 관리**: 회원 목록 / 가입 승인(대기 1건 배지) / **메뉴 권한 매트릭스**(부서×메뉴 접근표).

## 마무리 멘트 포인트
- 현재는 **프론트 mock**으로 동작 → 백엔드 REST 계약(`backend/README.md`)이 이미 1:1 정의되어 있어 **서버 연동만 남음**.
- 보안(비밀번호 해시·주민번호 암호화·서버측 인가)은 2단계에서 서버와 함께 강화 예정.
- 단일 HTML 프로토타입(v0.9.33) → **유지보수 가능한 컴포넌트 구조**로 재정비 완료.
