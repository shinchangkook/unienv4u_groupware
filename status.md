## 기술 스택 결정

| 항목 | 선택 | 이유 |
|------|------|------|
| 프론트 | React 18 + TypeScript + Vite | 빠른 HMR, 표준적, 시연 안정성 |
| 라우팅 | react-router-dom | 스펙의 패널→라우트 매핑 |
| 상태 | React Context (auth) + 로컬 | MVP 규모에 과함 없이 |
| 데이터 | mock API 계층(`src/mock/`) | 백엔드 없이 시연, REST 형태로 추상화 |
| 스타일 | 커스텀 CSS 변수(테마) | 원본 #185FA5 아이덴티티 계승, 라이트/다크 |
| 백엔드 | 계약(README/타입)만 1차 정의 | 2단계에서 구현 |
